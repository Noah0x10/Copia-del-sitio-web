Las 15 Mejores prácticas que los desarrolladores deberían saber

Este post está dedicado a aquellas personas que acaban de comenzar en el mundo del desarrollo web.
Debajo encontrarás 15 tips para escribir en HTML. Basicamente esta es la guía de buenas prácticas
para principiantes en HTML. La mayoría de páginas web usan HTML. Las prácticas mencionadas a
continuación no solo nos ayudarán a crear códigos limpios y bien estructurados sino que también
facilitarán la optimización del motor de búsqueda.Espero poder ayudarte a llevar a cabo la conversión de
PSD a HTML.

Cierra las etiquetas

Aunque parezca muy simple, sigue siendo una práctica muy común en la vida diaria. A veces, nosotros
omitimos etiquetas como "ol" o un "ul". Además, muchos eligen dejar de lado las etiquetas Li de cierre.
Es urgente abordar este tema, de lo contrario te encontrarás con problemas de validación y compatibilidad
todo ello relacionado a la codificación.

Usa palabras claves en los títulos de etiquetas

Una de las cosas más importantes que ayuda a la optimización del motor de búsqueda de los sitios web es el uso
de palabras clave en los títulos de etiquetas. Esto último ayuda a las páginas web a ser significativamente más
amigables para los motores de búsqueda. El texto escrito en el título de etiqueta está desplegado en la página de
resultados.

Describe las etiquetas meta

Aparte de los títulos de etiquetas, otra cosa importante que ayudarán en la optimización de los motores de búsqueda
de los sitios web es el uso de descripciones meta en la etiqueta meta. Cuando un navegante de internet escribe una
palabra clave en la barra de búsqueda las etiquetas meta al ser atributos, facilitan el que la página web destaque entre
otras hace que el motor de búsqueda le sea más fácil encontrar la página que se requiere, ¿Cómo funciona?, las
descripciones de las etiquetas meta dicen de qué trata la página. Describe brévemente el propósito de tu sitio web. Las
descripciones diferencian una página de otra. Necesitas escribir un texto preciso y diferente en la descripción de una
etiqueta meta por cada página web.

Usa palabras clave en los atributos meta

Estos atributos contituyen una lista de palabras claves importante y relacionados a tu página web con frases separadas
por comas. Estas palabras clave funcionan a favor en la creación de páginas web que destaquen o diferencien de otras.
No seas malvado, no hagas spam! muy amablamanete te pido que no pongas palabras claves solo para rellenar porque
eso no te va a ayudar. Basta con poner pocas palabras y frases que describan correctamente tu página web.

No uses el atributo style en líneas de textos

Hay tres formas para insertar una capa con el atributo style.Un style en una línea pierde muchas de sus ventajas al mezclar
contenido con presentación. Usa este método pocas veces!. Para usar style en líneas tu usas el atributo style en etiquetas
importantes. El atributo style puede contener cualquier proceso con CSS. El ejemplo muestra cómo cambia el color y el
margen izquierdo de un parrafo .

Puede verse bien, pero puede causar errores en prácticas de codeo.Siempre crea un archivo separado de CSS para tu atributo
styles. Solo evita hacer uso de style mientras estás codeando. Simplemenete, una vez que hayas terminado con la parte de la
codificación, podrías empezar añadiendo stilos.

Coloca el código de Java script en la parte inferior

Necesitas un lugar adecuado para el código JavaScript en una página hecha de HTML para estar seguros de que tu buscador lo
ejecute. No es demasiado difícil, pero tienes que separar lo que es JavaScript y HTML. Uno de los muchos parametros del
algoritmo de google para clasificar a los sitios web es el tiempo de carga del sitio web. Una de las muchas metas de todo
desarrollador web es reducir el tiempo de carga de los sitios web. Solo coloca los archivos de JavaScript (Si el propósito es solo
añadir funcionalidad) en la parte de abajo, solo antes de cerrar con la etiqueta body porque cuando cargan un texto, el buscador
no puede continuar hasta que se haya cargado todo el archivo. Los usuarior tendrán que esperar un largo tiempo hasta que aparezca
algo en pantalla

Divide tu diseño usando la etiqueta div

Usando la etiqueta div te permite definir la página en partes lógicas, no solo la localización de los diseños. Si estás usando la etiqueta
div en combinación con CSS, tu crearás una página que será más fácil manejar. Mayormente sucede que en una página web está
constituida por diferentes secciones con un contenido determinado, entonces usar esta etiqueta los divide, estarás listo para estilizarlos
como quieras. Usar excesivamente la etiqueta div es también una mala práctica. Algunos desarrolladores usan esta etiqueta para contener
más etiquetas div conteniendo más etiquetas div hasta crear una montaña de divs. Simplemente no tiene sentido usar esta etiqueta como
último recurso cuando no hay etiquetas más adecuadas.

Pon los nombres de las etiquetas en minúscula

Mantener el marcado en minúsculas es una buena práctica. Ponerlo en mayúsculas no causa ningún daño, pero afecta a la legibilidad del
marcado. Es bueno si mantienes todos los nombres de tus etiquetas en minúsculas. También es bonito para la vista.

Agrega el atributo alt en las imágenes

La etiqueta img inserta una imágen en una página HTML. Tiene dos atributos obligatorios:src y alt. El texto ALT debería estar compuesto
como una alternativa textual adecuada a la imágen: a veces eso puede acabar siendo una descripción de la imágen, pero en la práctica esta
decisión tiene mayor frecuencia en fallar que en acertar. Usar un atributo alt es bastante importante para escribir códigos semánticos y
válidos. Algunos quizás ignoraron esta parte, pero los desarrolladores necsitan llenar esta sección por razones de validación y accesibilidades.

Uso apropiado de los encabezados

Usar apropiadamente las etiquetas para mejorar tu sitio SEO. Las etiquetas de encabezado son elementos muy importantes en crear y mantener
amigable el motor de búsqueda del sitio web o blog. Hay 6 niveles de encabezado. Estos niveles son los siguientes: H1,H2,H3,H4,H5,H6. La
etiqueta de encabezado H1 tiene el tamaño de fuente más grande, mientras que el H6 tiene el más pequeño. El motor de búsqueda mira las etiquetas
de encabezados porque ellos formas la presentación para encontrar contenido, así como rutas en su blog o sitio web.

Terminar de navegar con la lista desordenada

Casi todos los sitios web/blogs tienen algún tipo de estado de navegación. No es necesario que uses la etiqueta div para terminar con la sección
de navegación. Mejor utiliza ul para terminarlo. La etiqueta ul está destinada a contener una lista de elementos.

Usa el CSS reinicio

Es bastante debatible acerca del uso del CSS reinicio. la mejor forma posible es desarrollar nuestro propio CSS reinicio. Quizás empieces con usar
algunos CSS reinicio populares como el de Eric Meyer, luego progresivamente puedes ir implementando tu propio stilo de acuerdo a lo que requieres.
Si tú no haces esto, no entenderás el por qué tu lista de elementos reciben ese poco espacio adicional cuando no lo especificaste en ninguna parte de tu
archivo CSS.

Aprende a usar bien los elemtnos de HTML en el lugar correcto

Con el tiempo, asegúrate de conocer todos los elementos de HTML. Úselos correctamente para un contenido estructural semántico y válido.

Por ejemplo.

Los < p >elementos que definen un parrafo en un documento de HTML.

El < body > elementos que definen el cuerpo de un documento en HTML.

< ul > (Listas desordenadas) cuando el orden de la lista de elementos no son importantes.

< ol > (Listas ordenadas) cuando el orden de la lista de elementos son importantes.

< dl >(Listas de definiciones) para pares de elementos/definiciones.

En lugar de usar < i > y < b >, usa < em > para enfatizar y < strong > para enfatizar mucho.

En lugar de usar < br > para colocar una línea entre parrafos, usa las propiedades de margen y/o relleno de CSS.

Todas estas cosas juegan un papel importante en la conversión de PSD a XHTML.

Ver fuente

Nadie se vuelve perfecto en el primer movimiento. Al inicio, quizás visites algunos buenos sitios webs y revisar cómo han codificado diferentes
secciones del sitio web. Incluso puedes buscar algún código en JavaScript utilizado por otras personas y puedes aplicarlo de manera similar a tu
trabajo. Mira la fuente y busca la etiqueta < head > el nombre del script. Es bueno aprender de otros pero no sigas la mala práctica de robar el
diseño de los demás.

Validación del código

Necesitas validar tu marcado continuamente. Bueno! No es como que un marcado el cual no está terminando de validar sea incorrecto, y que esté
completamente validado es una gran pieza de trabajo

Pero aún así necesitas descargar W3C validación de marcado (W3C markup validation) o barra de herramientas para desarrolladores de Firefox
(Firefox developers tool bar) para la validación. Buscará errores, si los hay. Haz la validación con frecuencia, de esta forma puedes evitar varios
problemas que son díficiles de resolver.
